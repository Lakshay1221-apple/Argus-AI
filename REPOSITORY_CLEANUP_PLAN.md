# Repository Cleanup Plan: Argus AI

This report outlines the structural cleanup audit of the **Argus AI** repository. Every file has been cataloged and evaluated to define a minimal, clean, and highly reproducible layout for active development and long-term maintenance.

---

## 🏛️ Classification Matrix

Each file in the repository is classified into one of the following statuses:
*   **KEEP**: Active production code, configuration, dataset, or utility essential for execution or training.
*   **ARCHIVE**: Obsolete or historical dataset/report to be moved into a local `_archive/` subdirectory.
*   **DELETE_CANDIDATE**: Redundant, empty stub, or obsolete files safe to delete.
*   **REVIEW**: Files with configuration anomalies (like filename spaces) or empty directories requiring a maintainer decision.
